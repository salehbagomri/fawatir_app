# خطة بناء تطبيق الفواتير — وثيقة الوكيل المخطّط

> وثيقة موجّهة للوكيل المنفّذ في محرّر الكود. تُنفَّذ **مهمة واحدة في كل مرة**، مع التحقق من معايير القبول قبل الانتقال للمهمة التالية.

---

## 1. نظرة عامة

تطبيق Flutter شخصي لإدارة فوترة اشتراكات (ستارلينك وغيرها) لعدة جهات: عملاء، فواتير، تحصيلات، سندات قبض، كشوف حساب، واشتراكات شهرية متكررة. قاعدة بيانات محلية مع نسخ احتياطي إلى Google Drive.

### القرارات المثبّتة (لا يُعاد التفاوض عليها)
- **بلا ضريبة (VAT).**
- **عملة لكل عميل** (`accountCurrency`) — كشف الحساب والرصيد بها.
- **دعم اختلاف العملة:** الدفع قد يكون بعملة غير عملة الفاتورة؛ المقارنة دائماً بعد التحويل لعملة حساب العميل عبر `fxRateToAccount`.
- **جهاز واحد، قاعدة محلية** + نسخ احتياطي/استعادة عبر Drive، مع تصدير/استيراد يدوي يعمل أوفلاين بالكامل.
- **نظام الرصيد المتدحرج** (balance-forward) للحساب، مع دعم ربط الدفعة بفاتورة لتحديث حالتها.
- المبالغ تُخزّن كـ **وحدات صغرى** (int، المبلغ × 100).

### الحزمة التقنية
Flutter • Drift (SQLite) • Riverpod • GoRouter • pdf + printing • share_plus • google_sign_in + googleapis (Drive) • path_provider • file_picker • intl • google_fonts.

### الملفات الجاهزة (تُدمج كما هي مع تعديلات طفيفة)
| الملف | الوجهة في المشروع | الدور |
|---|---|---|
| `invoice_schema.dart` | `lib/data/db/tables.dart` | جداول Drift |
| `invoice_pdf.dart` | `lib/shared/pdf/invoice_pdf.dart` | توليد فاتورة PDF |
| `receipt_pdf.dart` | `lib/shared/pdf/receipt_pdf.dart` | توليد سند قبض PDF |
| `payment_logic.dart` | `lib/features/payments/data/payment_repository.dart` | تسجيل التحصيل وتحديث الحالة |

## 2. طريقة العمل مع المنفّذ

1. مرّر للمنفّذ **خطوة واحدة** بصياغة: «نفّذ T#.# فقط، ثم توقّف لأتحقق».
2. بعد كل مهمة: شغّل `flutter analyze` و`dart run build_runner build` عند الحاجة، وتأكد من معيار القبول (DoD).
3. لا تنتقل لمرحلة جديدة قبل اكتمال سابقتها (إلا حيث يُذكر أنها مستقلة).
4. أي قرار تصميمي جديد يخرج عن هذه الوثيقة → ارجع للمخطّط (أنا) قبل التنفيذ.

### بنية المجلدات المعتمدة (feature-first)
```
lib/
  main.dart
  app/            router.dart · theme.dart · app.dart
  core/           money.dart · fonts.dart · result.dart · constants.dart
  data/db/        tables.dart · database.dart · database.g.dart
  features/
    company/      (data · application · presentation)
    clients/
    invoices/
    payments/
    statements/
    subscriptions/
    backup/
    dashboard/
  shared/
    pdf/          invoice_pdf.dart · receipt_pdf.dart · receipt... 
    widgets/
```
كل feature ثلاث طبقات: `data/` (repository) · `application/` (Riverpod providers/controllers) · `presentation/` (شاشات وودجت).

## المرحلة 4 — الفواتير

### T4.1 ترقيم الفواتير
- توليد الرقم من `invoicePrefix` + السنة + عدّاد متزايد ذرّي (تحديث `invoiceCounter` ضمن معاملة Drift).
- ✅ تم عندما: فاتورتان متتاليتان تحملان رقمين فريدين متتاليين.

### T4.2 إنشاء/تعديل فاتورة
- اختيار العميل، التاريخ، الاستحقاق، العملة (افتراضها عملة العميل)، `fxRateToAccount` (يظهر فقط إذا اختلفت عن عملة الحساب).
- بنود ديناميكية (وصف، كمية، سعر) مع حساب الإجمالي لحظياً.
- حفظ الفاتورة وبنودها في معاملة واحدة، وتحديث `totalMinor`.
- ✅ تم عندما: فاتورة بثلاثة بنود تُحفظ بإجمالي صحيح.

### T4.3 قائمة وتفاصيل الفاتورة
- قائمة بفلترة الحالة، وتفاصيل تعرض البنود والإجمالي والحالة (مع «متأخرة» عبر `isInvoiceOverdue`).
- ✅ تم عندما: تظهر الفاتورة بكل تفاصيلها وحالتها الصحيحة.

### T4.4 توليد ومشاركة PDF
- اربط `invoice_pdf.dart`: حوّل بيانات الفاتورة إلى `InvoicePdfData` (اقرأ بايتات الشعار من المسار)، ثم `buildInvoicePdf` ← `shareInvoicePdf` / `previewInvoicePdf`.
- فعّل نسخة الخط المحلّي (assets) في `_loadFonts` للعمل أوفلاين.
- ✅ تم عندما: زر «مشاركة» يفتح واتساب بملف PDF عربي صحيح.
